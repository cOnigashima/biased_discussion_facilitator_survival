# Bias Meeting Game Handoff

ソフトウェア開発組織を舞台にした、探索パート + 会議バトル型ADV / 推理ゲームの企画メモです。次のセッションやCodexに渡して、そのままプロトタイプ設計・実装・シナリオ作成に進めることを目的にしています。

## 現時点の方向性

プレイヤーは、開発プロジェクト内の会議・レビュー・障害対応・顧客折衝などに参加する。探索パートでSlack、Jira、PR、CIログ、障害ログ、顧客メール、過去議事録などを読み、証拠カード・過去発言カード・人物の利害・隠れた前提を集める。バトルパートでは、会議中の発言や議事録に対して、証拠・質問・共感・フレーム変換・退路提示などを使い、会議の流れを変える。

目指す手触りは、逆転裁判的な「発言を突く」快感、ダンガンロンパ的な「発言にタイミングよく撃つ」テンポ、Detroit: Become Human的な「状態値・フラグ・分岐図」を、ソフトウェア開発現場の意思決定に寄せたもの。

ただし犯人探しではない。会議で誤った前提が固まる前に止め、より現実的な判断案を通すことを主な遊びにする。

## 重要な前提

- バイアス名当てクイズにしない。
- 「それは正常性バイアスです」のようなラベル貼りは、多くの場合ゲーム内では悪手にする。
- 楽しいのは、発言・議事録・証拠・利害を読んで、「その前提はまだ確定していない」と会議の流れを変えること。
- キャラを単なる愚か者にしない。実績、責任、恐れ、守りたいものを持たせる。
- ソフトウェア開発現場の具体物を重視する。PR、CI、Feature Flag、ロールバック、顧客スコープ、Go/No-Go、ポストモーテムなど。
- プロダクト化を見据えるが、面白さの核はまだ仮説段階。検証仮説を残しておく。
